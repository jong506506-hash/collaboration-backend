# 업무 연속성 관리 서비스 MVP 정리

## 1. 서비스 목적
- 담당자가 회의, 휴가, 퇴사, 자리비움 상태여도 업무가 멈추지 않도록 만든다.
- 특정 사람에게만 쌓여 있는 업무 맥락을 문서와 기록으로 남겨 신규 인력도 빠르게 이어받을 수 있게 한다.
- 질문, 답변, 의사결정, 인수인계 내용을 검색 가능한 자산으로 축적한다.

## 2. 핵심 문제 정의
- 담당자 부재 시 타 팀이나 신규 담당자가 누구에게 물어봐야 하는지 알기 어렵다.
- 업무 진행 맥락이 메신저나 통화에만 남아 있어 후속 처리와 인수인계가 어렵다.
- 반복 질문이 누적되지만 정리된 문서나 FAQ로 전환되지 않아 비효율이 커진다.
- 특정 업무를 한 사람만 아는 경우 퇴사나 조직 변경 시 업무 연속성 위험이 커진다.

## 3. 주요 사용자
- 실무 담당자
- 팀장 또는 파트 리더
- 타 부서 협업자
- 신규 입사자 또는 업무 이관 대상자
- 운영 관리자

## 4. MVP 범위
### 4-1. 담당자/조직
- 사용자 등록
- 팀 등록
- 사용자 상태 등록
- 대체 담당자 지정

### 4-2. 업무
- 업무 카드 등록
- 주 담당자/부 담당자/협업자 지정
- 업무 상태와 우선순위 관리
- 인수인계 필요 여부와 지식 편중 위험 표시

### 4-3. 문서/지식
- 인수인계 문서 작성
- 업무 매뉴얼/FAQ/결정 로그 문서화
- 관련 시스템, 후속 액션, 주의사항 기록
- 문서 최신성 확인 기준 저장

### 4-4. 질문/응답
- 질문 스레드 등록
- 대상 사용자 또는 대상 팀 지정
- 담당자 부재 시 라우팅 대상자 확인
- 답변 채택 및 해결 요약 저장

### 4-5. 알림/검색
- 부재 등록 알림
- 질문/업무 위험 알림
- 업무명, 담당자, 팀, 키워드, 시스템명 기반 검색

## 5. 확장 기능
- 회의록/통화 요약 자동 생성
- 인수인계 문서 초안 생성
- 유사 질문/유사 문서 자동 추천
- 오래된 문서 경고 자동화
- 업무 위험도 점수화
- 신규 입사자 온보딩 경로 추천

## 6. 대표 화면
- 대시보드
- 담당자 상태 보드
- 업무 목록/상세
- 질문 센터
- 인수인계 센터
- 지식 문서 검색
- 관리자 설정

## 7. 백엔드 도메인 초안

- `collab_user`
- `collab_user_status`

- `collab_question_thread`

- `collab_notification`

## 8. 구현 순서 제안
1. 팀/사용자/상태/업무 엔티티와 기본 API 구성
2. 인수인계 문서와 지식 문서 작성/조회 API 추가
3. 질문/답변/라우팅 흐름 구현
4. 대시보드와 검색 API 추가
5. AI 추천/요약 기능 확장
